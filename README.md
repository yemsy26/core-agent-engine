# Core-Agent-Engine

A high-performance, asynchronous agent execution runtime and deterministic security firewall implemented natively in Rust. Built on top of the `tokio` asynchronous stack, this engine acts as a secure, low-latency sandbox for autonomous AI agents requiring system-level integration (Tool Calling/Bash execution) without compromising host integrity.

By enforcing a **Secure-by-Default** paradigm, `core-agent-engine` eliminates prompt injection exploits, system mutation risks, and "success hallucinations" commonly found in traditional Python-based agent frameworks.

---

## 🛡️ Core Architectural Columns

### 1. Deterministic Permission Firewall
Every system payload generated by an LLM tool-call is intercepted before process spawning. The runtime evaluates commands against compiled Regular Expression patterns defined in a strict `PermissionPolicy` (`allow`/`deny` registers). If a command matches a deny rule or fails to match an explicit allow rule, the engine halts execution immediately, preventing destructive payload mutations (e.g., malicious script execution or unauthorized network bindings).

### 2. Active Physical Verification Pipeline
To solve the industry-wide issue of **agent hallucination** (where an AI assumes a command succeeded based on ambiguous console outputs), `core-agent-engine` implements a post-execution physical validation layer. The runtime cross-references filesystem mutations using native `std::fs` primitives to physically confirm changes on the host system before returning a successful execution token to the agent.

### 3. Glass-Box Event Streaming (Asynchronous Auditing)
Designed for enterprise telemetry compliance, the firewall incorporates non-blocking asynchronous channels via `tokio::sync::mpsc`. The runtime streams real-time auditing tokens (`SecurityEvent::Allowed` / `SecurityEvent::Denied`) during every evaluation block, allowing external monitoring sub-systems to track agent behavior with zero impact on the core logic latency.

### 4. Low-Latency Rust Architecture
By removing Python runtime interpretation overhead and heavy virtual environment layers, this engine provides microsecond response times. It is optimized to interface seamlessly with local LLM infrastructures (such as **Ollama** or **Llama.cpp** local sockets), making it perfect for embedded or edge computing infrastructure.

```text
         ┌───────────────────────────┐
         │ PermissionPolicy Firewall │

                       │
             ┌─────────┴─────────┐

      ▼             ▼

```

## 💻 Developer Guide & Usage Example

The following production-grade snippet demonstrates how to initialize the secure runtime environment, configure the permission policies, stream auditing events, and safely execute system payloads:

```rust
use core_agent_engine::{
    security::{PermissionPolicy, PermissionRule, ToolKind, SecurityEvent},
    tools::bash::BashCommand,
};
use regex::Regex;
use tokio::sync::mpsc;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // 1. Initialize the asynchronous, non-blocking telemetry channel
    let (tx, mut rx) = mpsc::channel::<SecurityEvent>(100);

    // 2. Formulate strict security rules via Regular Expressions
    let allow_rule = PermissionRule {
        tool: ToolKind::Bash,
        pattern: Regex::new(r"^mkdir -p /tmp/agent_sandbox/.*")?,
    };

    let deny_rule = PermissionRule {
        tool: ToolKind::Bash,
        pattern: Regex::new(r".*rm -rf.*|.*chmod.*")?,
    };

    // 3. Build the Deterministic Firewall Policy
    let policy = PermissionPolicy::new(vec![allow_rule], vec![deny_rule], tx);

    // 4. Background task to ingest and stream real-time auditing events
    tokio::spawn(async move {
        while let Some(event) = rx.recv().await {
            match event {
                SecurityEvent::Allowed(cmd) => println!("[AUDIT LOG - ALLOWED]: {}", cmd),
                SecurityEvent::Denied(cmd) => eprintln!("[AUDIT LOG - BLOCKED]: WARNING: Unauthorized access attempt: {}", cmd),
            }
        }
    });

    // 5. Simulate an agent payload execution target
    let agent_command = "mkdir -p /tmp/agent_sandbox/workspace";
    
    println!("Evaluating agent payload...");
    match policy.enforce(ToolKind::Bash, agent_command).await {
        Ok(_) => {
            // Policy cleared the command -> Delegate execution to the secure runtime
            let runner = BashCommand::new(agent_command, "/tmp/agent_sandbox/workspace");
            let output = runner.execute().await?;
            println!("Execution completed successfully. Physical verification validated.");
        }
        Err(e) => {
            eprintln!("Execution aborted by the firewall runtime: {:?}", e);
        }
    }

    Ok(())
}

```
